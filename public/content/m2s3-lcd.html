IÂ²C is a master-slave, multi-master bus. The Arduino acts as the master, sending requests to slave devices (the LCD). Each slave device has a unique address (e.g., 0x27 or 0x3F).

  • **SCL (Serial Clock):** Generated by the master (Arduino) to synchronize data transfer.
  • **SDA (Serial Data):** Carries the actual data being sent to or read from the slave device.
  • The I2C backpack handles the translation of the two-wire signal into the 16 parallel pins needed by the LCD module.

Find the correct address for your I2C module (usually 0x27 or 0x3F).


#include <LiquidCrystal_I2C.h>

// Set the LCD address (check your module)
LiquidCrystal_I2C lcd(0x27, 16, 2); 

void setup() {
  lcd.init();      
  lcd.backlight(); // Turn on the backlight
  lcd.print("Robotics Arm");
}

void loop() {
  lcd.setCursor(0, 1); // Column 0, Row 1
  lcd.print(millis() / 1000); // Display seconds since start
  lcd.print(" s");
  delay(100);
}
